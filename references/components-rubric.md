# 可复用写作组件 Rubric（components.md 怎么写）

`analyze.py` 给出的是**描述性统计**（词频、标题模式、结构、金句候选）。
把这些统计 + 通读原文后的判断，提炼成「拿来就能用」的写作组件，写进 `components.md`，
再用 `render_dashboard.py <analysis.json> components.md dashboard.html` 重渲染嵌入看板。

**底线**：抽的是**可复用的骨架**（怎么搭），不是作者的观点/经历/资源（不抄答案）。

## 七个组件（对齐原版）

### 1. 选题脚手架
3-5 个可套用的选题角度公式，例：
- `我用 {工具} 做了 {成果}，附 {实操/源码}`
- `{热点事件} 出来了，我实测后发现 {反常识结论}`

### 2. 标题公式
从 `title_analysis` 的高频模式反推 3-5 个填空式标题模板，例：
- `{数字}个 {领域} {工具}，第 {N} 个我天天用`
- `{动作动词}+{对象}，我把 {过程} 全开源了`

### 3. 开头模板
2-4 种钩子开头骨架（数字钩 / 反差钩 / 提问钩 / 故事钩），各给一句填空式示范。

### 4. 段落句式
基于结构统计（平均段落、长短句习惯）总结 3-5 条可复用句式节奏，例：
- 短段落连续推进，每段 1-2 句；
- 口语停顿词（「好家伙」「说实话」）做节拍。

### 5. 转场句
5-8 个可直接套用的承上启下短句模板（「说完 X，再聊聊 Y」式）。

### 6. 金句生成器
从 `punchlines` 候选里归纳 3-5 个句式模子（下定义 / 升维 / 对仗反转），
每个模子给「公式 + 1 个新造示例」。新造示例要标注「（仿写，非原文）」。

### 7. 发布前检查清单
8-12 条 checklist，覆盖：标题钩子是否够强、开头 3 秒留人、案例是否核实、
配图节奏、金句密度、结尾行动号召、错别字、合规。

## components.md 推荐结构
```
## 选题脚手架
## 标题公式
## 开头模板
## 段落句式
## 转场句
## 金句生成器
## 发布前检查清单
```
用 `##` 二级标题分节（md_min 渲染器支持 #/##/###、**加粗**、`代码`、有序/无序列表）。
