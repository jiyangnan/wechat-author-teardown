# 作者方法论拆解 Rubric

读完归档的全部文章后，按下面六个维度拆解作者的方法论，输出一份 `methodology-report.md`。
**目标是「学方法、不抄答案」**：拆的是作者怎么判断、怎么搭结构、怎么复盘，不是搬运他的观点、经历、资源。

## 六大拆解维度

### 1. 选题系统（Topic System）
- 这个号的选题集中在哪些主题簇？（用归档标题做聚类）
- 选题的共同触发器是什么？（蹭热点 / 工具实测 / 踩坑复盘 / 行业判断 / 资源整合）
- 选题的「时效 vs 长青」配比？
- 给出 3-5 个可复用的**选题角度模板**（如「我用 X 做了 Y，附实操」）。

### 2. 分析框架（Analysis Framework）
- 文章里反复出现的思考结构是什么？（问题→方案→验证→结论 / 现象→本质→对策 …）
- 作者怎么把一个复杂话题拆成读者能跟上的台阶？

### 3. 判断标准（Judgement Criteria）
- 作者凭什么说一个东西「值得 / 不值得」？他的评价坐标系是什么？
- 他怎么在真实任务里做取舍？（这是最值得学的部分，重点记录）

### 4. 表达模板（Expression Templates）
- 开头怎么起？（钩子类型：数字 / 反差 / 提问 / 故事 / 痛点）
- 段落节奏：长短句配比、是否大量短段、是否爱用「好家伙」式口语停顿。
- 转场怎么接？结尾怎么收（升华 / 行动号召 / 留钩子）？

### 5. 案例库（Case Bank）
- 作者常引用哪类案例？（自己的项目 / 开源项目 / 名人 / 数据）
- 案例的真实性与可验证性如何？（标注是否有原始出处）

### 6. 金句风格（Punchline Style）
- 摘录 10-15 条代表性金句。
- 总结金句的句式规律（对仗 / 反转 / 下定义 / 升维）。

## 报告输出结构（methodology-report.md）

```
# {账号} · 方法论拆解报告

> 样本：{N} 篇 · 时间跨度：{最早}~{最新} · 拆解时间：{date}

## 一图速览
（一句话定位这个号 + 三个最突出的特征）

## 1. 选题系统
## 2. 分析框架
## 3. 判断标准
## 4. 表达模板
## 5. 案例库
## 6. 金句风格

## 可复用清单（拿来就能用）
- 选题角度模板 ×N
- 标题公式 ×N
- 开头钩子模板 ×N
- 金句句式 ×N

## 学习边界（必须保留这一节）
明确：可学的是判断/验证/结构/复盘；不可抄的是经历、资源、价值观、表达习惯。
```

## 底线
- 不捏造作者没表达过的观点。
- 金句必须是原文摘录，标注出处文件。
- 报告是「学习材料」，不是「洗稿模板」。
